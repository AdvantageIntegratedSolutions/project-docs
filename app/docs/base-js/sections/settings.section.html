Settings Initialize your client using these settings in your configuration object.

When initializing a client, base-js takes an object of settings to help enhance its capabilities. Below are the object properties available:

Parameter Value Required
async 'callback' | 'promise' | 'false' yes
apptoken Application token string. yes, if the application requires application tokens and no usertoken is supplied.
database id QuickBase database ID yes
tables Object of table ids and field ids yes
ticket Valid authentication ticket. no
username Valid quickbase username. no
password Valid quickbase password. no
usertoken Valid API user token. This value takes precedence over username/password and ticket. no
ticket Valid authentication ticket. no
realm Quickbase realm. no
Tables Control your tables with predefined table names, field names and id's.

The tables parameter is an important required setting. This parameter is an object of all the tables that will be used in your application. This includes the table name, table id's, field name and field id's.

    
      const tables = {
        toys: {
          dbid: "bjbsgxy3t",
          rid: 3,
          name: 6,
          type: 8,
          description: 9,
          voicedBy: 10,
          primary: 11
        },

        locations: {
          dbid: "bjbsgxy83",
          rid: 3,
          name: 6,
          address: 7
        }
      };
Table names and field names can be named whatever you like. No need to match anything to QuickBase. These names will be used throughout the life of your project including the request and response data. Async Options Promises, callbacks or synchronous http requests

When you execute something synchronously, you wait for it to finish before moving on to another task. When you execute something asynchronously, you can move on to another task before it finishes. Below are the different options available for the async setting parameter:

The 'promise' and 'callback' setting tell the library to make all HTTP requests asynchronously. 'false' will execute all API requests synchronously.(not recommended)

Value Description
promise All HTTP requests will be executed asynchronously and will return a promise.
callback All HTTP requests will be executed asynchronously and a callback function will be executed once the request is complete.
false All HTTP requests to QuickBase will be made synchronously.
Using promises should be your #1 option. Callbacks #2. Synchronous is #3 but discourged and will most likely be deprecated in future versions.
