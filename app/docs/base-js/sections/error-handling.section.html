Error Handling

As of v4.9.0, all HTTP errors or QuickBase errors are returned.

Errors will bypass any callback functions and automatically return an error JSON object. A message description of the error will be included along with a error code. The error code will either reflect a network error code or the actual QuickBase error codes.

Examples
  	
  		{ error: { message: "Something went wrong.", code: "500" } }
  		{ error: { message: "Invalid input", code: "2" } }
  		{ error: { message: "Missing required field", code: "50" } }
Please provide feedback on error handling as neccessary as this is a new as of v4.9.0 and may not meet all needs.
