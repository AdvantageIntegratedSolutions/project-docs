Introduction

base-js is a platform to develop and store documentation for common libraries, projects and tools.

project-docs provides a standard UI to improve ease of use and readability, tools to shorten the documentation process and help develop documentation standards.

All documentation is written in common HTML & SASS, no new rules or markdown to learn.

Functionality includes helper tags, responsive CSS framework, automatic side-navigation and syntax highlighted code examples.

Documented Projects base-js quickstart Document New Project Documenting a new project is easy. Steps to document a new project
  1. Add a new html file for your project to the docs folder.
  2. Add the name of the new document to the "_docs.js" file.
  3. Add the new project to the project-docs list of current projects("project-docs.doc.html")
 Doc Style Guide

All documentation is written in common html. No new markdown language or a long list of rules. Be sure to take advantage of the responsive CSS framework included(Materialize).

Helper Tags

Helper tags help speed up the process of writing the HTML by packing up the common parts of a document.

doc-section Adding class to headers will add heading to side-navigation.
doc-header Adding class to headers will add heading to side-navigation.
doc-sub-header Adding class to headers will add heading as indented header to side-navigation.
doc-summary Adding class to headers will add heading as indented header to side-navigation.
doc-code Adding class to headers will add heading as indented header to side-navigation.
doc-notice Adding class to headers will add heading as indented header to side-navigation.
doc-alert Adding class to headers will add heading as indented header to side-navigation.
doc-caption Adding class to headers will add heading as indented header to side-navigation.
doc-card Adding class to headers will add heading as indented header to side-navigation.
These tags are optional. You can use any HTML tag you wish but these tags pack a punch. Classes

These classes apply common styling or bind functionality to your elements as usual.

doc-nav-hide Added to 'doc-header' or 'doc-sub-header' will prevent the header from displaying in side-nav.
Side-Navigation The side-navigation is automatically rendered.

To add a header to the side-navigation, add a 'doc-header' or 'doc-sub-header' tag to your document. The side-navigation will be rendered in the order the headers appear in your document. Adding the class, "doc-nav-hide", will prevent the header from displaying in the side nav.

Ensure to add an 'id' attr to your headers and subheaders to create an anchor. Place the headers and subheaders in order. It doesn't make sense to put a subheader before a header or have a subheader without a header. Deployment

Just push to git. Application is hosted using Github pages.
