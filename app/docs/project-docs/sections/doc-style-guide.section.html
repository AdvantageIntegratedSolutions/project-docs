Doc Style Guide

All documentation is written in common HTML and SASS. No new markdown language or a long list of rules. Be sure to take advantage of the responsive CSS framework included(Materialize).

Organization Guide

To create documentation that is consistent across projects and take advantage of new reusable components, below is a list of general rules to structure and organize your documentation.

Sections For each section of documentation, create a new section file in your sections project folder and include it in your index file. This will increase maintainability over time and keep files short and sweet. Wrap all data in section with a 'doc-section' component tag.
Headers Use clear and concise header names. Headers automatically render in the side-navigation so they must be simple and make their purpose obvious. Wrap headers with a 'doc-header' component tag so they are automatically loaded in the side-nav.
Misc Content Store content in 'p' tags.
Collections Use tables to store collections. Use the 'table' tag and include the 'table', 'striped', 'bordered' classes provided by Materialize.
Notices Use notices when you need to draw attention to important information. Use the 'doc-notice', 'doc-warning', or 'doc-best-practice' component tags.
Code Examples Include simple and direct code examples whenever possible. Use the 'code' tag wrapped with a 'pre' tag to display syntax highlighted code. The syntax will try to be automatically detected but to set the language explicitly include the class such as 'language-javascript', 'language-html' or 'language-css'. here is a list of all supported languages
Components

Utilize these components to reuse common elements of documentation:

doc-section Use to separate sections of documentation.
doc-header Use to add new header to the side-nav and create the anchor.
doc-caption Pair with doc-header to create caption under header.
doc-notice Use to build notice on page.
doc-warning Use to build warning on page.
doc-best-practice Use to build best-practive notice on page.
These tags are optional. You can use any HTML tag you wish but these components pack a punch. Classes

These classes apply common styling or bind functionality to your elements as usual.

secondary Add this class to 'doc-header' to have header display as a subheader in the navigation pane.
table Add common styling to a table
striped Add row striping to your table
bordered Add borders to your table
