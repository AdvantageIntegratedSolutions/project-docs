Codebase Changes that need to be made inside of the codebase for the form-engine to get new clients up and running
Update files: config/quickstart.config.js & config/schema.service.config
  • config/quickstart.config.js
    • convert table DBID's over to their new counterparts

appConfig changes
  • DON'T change name (this is the prefix for the page that is built out with the gulp task)
  • description shouldn't be visible so shouldn't matter
  • CHANGE Client to match new client's desires. If there is no client Display name then the client object will be displayed. If there is a client display name this is what will be displayed over the client
  • DON'T change username (this is how the interaction with the lambda works)
  • origin and authors are fine empty
  • DON'T change the bootstrap
  • CHANGE timezone to match clients needs (check BaseJS docs for syntax accepted)
  • baseConfig
    • DON'T change quickstart from true
    • CHANGE realm to match clients quickbase realm (Will be last step after transferring app to client)
    • DON'T change async from 'promise'
    • databaseID will need to be changed to match the new ids instructions where to find this is under the get Table DBID's section
  • the formBuilder[useNewDownload] object should remain true for all new clients (only legacy clients rely on a false value)
  • userManagement will need to be updated with the link relating to the appropriate link (which will basically cover changing the realm, dbid, and potentially the pageID#)
  • If manual reference to table (for example AIN-portal) is needed add table information inside of the quickstart.config.js inside tables object under databaseID

SchemaService does not currently function with applications with AppTokens
Make sure there is no required AppToken for form engine to work

When adding a table make sure to add permissions to Quickstart User role under users/manage-roles roles/quickstart permissions
  • ADD/DELETE = Checked
  • Fields = Full Access
  • View/Modify = All Records
  • Save Common Reports = unchecked
  • Edit Field Properties = unchecked
  • under App Access only Edit app structure and permissions is checked, no others should be checked
